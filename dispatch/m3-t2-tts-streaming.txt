# M3-T2: TTS 流式开口 — Codex 施工 Prompt

## 目标

在 Vela Electron 应用中接通 MiniMax WebSocket TTS，让语音模式开启后，Vela 的回复能**流式出声**（不等全部文字生成完才开始说话）。

## MiniMax API Key

用于 TTS 的 MiniMax key 和 LLM 用的是同一个，已经在 `vela.jsonc` 的 `llm.apiKey` 字段里了。你可以在 config 层让 TTS 复用同一个 key，或者在 `vela.jsonc` 的 `tts` 块里加一个 `apiKey` 字段引用同一个值。

## 音频播放方案

MiniMax WS TTS 返回的音频格式由 `vela.jsonc` 的 `tts.audioSettings` 控制，当前配置是：
- format: mp3
- sampleRate: 32000
- bitrate: 128000
- channel: 1

音频 chunk 通过 `speech-audio-chunk` 事件传到 renderer，每个 chunk 包含 base64 编码的 mp3 片段。

建议方案：
1. 在 renderer 端创建一个 `useAudioPlayer` hook 或 `AudioPlayerService` 类
2. 接收 `speech-audio-chunk` 事件，将 base64 解码为 ArrayBuffer
3. 用 Web Audio API（`AudioContext` + `decodeAudioData`）或 HTML5 `<audio>` + MediaSource Extensions 播放
4. 支持流式播放：收到一个 chunk 就开始播，不等全部收完
5. `speech-finished` 时标记结束
6. `speech-error` 时停止播放
7. 用户点"先停一下"（interrupt）时停止播放

**注意**：mp3 chunk 流式播放最简单的方式是用 MediaSource API，或者把每个 chunk 作为独立的小 mp3 文件依次排队播放。推荐后者，因为 MiniMax WS 返回的每个 chunk 通常是完整可解码的 mp3 片段。

## 验收标准

1. `npm run build` 通过
2. `npm run smoke` 通过
3. 启动 Electron 窗口 → 点击"回应方式"切到语音模式 → 发送一条消息 → 能听到 Vela 用 MiniMax TTS 的 Sweet Lady 声音流式回应
4. 切回文字模式后不再出声
5. 播放中点"先停一下"能中断语音
