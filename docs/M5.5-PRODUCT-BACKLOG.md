## M5.5 产品补全 — 任务清单

### Tier 1（不做就不能叫产品）

#### M5.5-T1 ASR 语音识别接入
- 优先级：P0
- 状态：TODO
- 说明：当前 `asr.enabled: false, provider: "placeholder"`。用户按语音按钮无法语音转文字，只有 TTS 单向输出。
- 方案方向：评估 Web Speech API（浏览器原生免费）/ Whisper API / MiniMax ASR，优先免费方案
- 验收：语音按钮按下 → 实时识别 → 文字填入输入框 → 发送

#### M5.5-T2 设置界面
- 优先级：P0
- 状态：TODO
- 说明：API key / 模型 / 语音 / 头像路径全靠手改 vela.jsonc，普通用户无法使用
- 方案方向：Electron 内嵌设置页（齿轮图标触发），覆盖 LLM provider + API key + TTS 开关 + 头像路径 + 用户位置
- 验收：设置修改后热重载生效，无需重启 app

#### M5.5-T3 错误处理 UI
- 优先级：P0
- 状态：TODO
- 说明：LLM 超时、网络断线、API key 错误、TTS 连接失败等场景，用户无可见反馈
- 方案方向：消息区内嵌错误提示卡片（红色/橙色），可重试按钮，不弹系统 dialog
- 验收：断网 / 错误 key / provider 超时三种场景都有友好提示

#### M5.5-T4 首次体验引导
- 优先级：P0
- 状态：TODO
- 说明：新用户打开后需要配 API key 才能聊天，无引导直接卡死
- 方案方向：首次打开检测无 API key → 引导设置页 → 填完后自动进入聊天
- 依赖：M5.5-T2 设置界面
- 验收：全新用户从安装到第一句对话的完整路径无卡点
